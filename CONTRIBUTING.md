## How to Contribute

### Reporting Bugs
- Report bugs using GitHub Issues.
- Provide a clear description of the bug and steps to reproduce it.
- Include your Node.js version, Docker version, browser, and OS.

### Suggesting Enhancements
- Suggest new features or enhancements using GitHub Discussions or by creating an issue with the `enhancement` label.

### Submitting Pull Requests
1.  Fork the repository.
2.  Create a new feature branch: `git checkout -b feat/your-feature-name`.
3.  Make your changes. Keep commits small and descriptive.
4.  Test your changes locally (`node Backend/server.js`) before submitting.
5.  Open a Pull Request against the `main` branch with a clear description of your changes.

### Code Style
- Use plain JavaScript (no TypeScript).
- Maintain consistency with `require` or `import` statements.
- Use 2-space indentation and single quotes.
